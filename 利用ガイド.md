# じょーず 利用ガイド（はじめて使う方向け）

この資料は、**じょーず**（SNS学習コーチング向けモバイルWebアプリ）を初めて開いた人向けです。  
**各タブでできること**と、**操作のタイミングで起きる挙動**を整理しています。

---

## 1. このアプリについて

- **目的**: 投稿の進行管理、コーチングセッションに紐づくタスク、スケジュール、目標の記録をまとめて扱います。
- **デモ版のデータ**: ブラウザの **localStorage** に保存されます。別の端末・別ブラウザとは共有されません。キャッシュ削除で消える場合があります。
- **画面の見た目**: **生徒**ログイン時はネイビー×ゴールド（SnsClub系）、**運営ロール**（マネージャー／リーダー／管理者など）ではレッド系（Levela系）に切り替わります。

---

## 2. ログイン

| 操作 | 挙動 |
|------|------|
| ログイン画面で **ロール別のクイックログイン** をタップ | そのロールのデモユーザーで入ります（パスワードは画面の案内どおり）。 |
| メール・パスワードを入力してログイン | 登録済みユーザーとして入ります。 |
| ログイン成功 | アプリ本体が開き、**ホーム**が表示されます。 |

---

## 3. 生徒向け：下部ナビの構成

生徒でログインすると、次のタブが表示されます（**投稿・ロードマップ・スケジュール**は生徒専用表示）。

| タブ | 主な用途 |
|------|-----------|
| **ホーム** | 今日の予定・学習の進み・投稿のサマリー・目標カードなど |
| **投稿** | 1本ずつの「投稿カード」と工程（台本・撮影など）の管理 |
| **ロードマップ** | マイルストーン・スターター・**セッションタスク** |
| **スケジュール** | カレンダー／メモ |
| **マイページ** | プロフィール・実績・投稿ペース・定期タスク・目標編集 |

> **補足**: 画面コード上には「今日の記録」でタスク回数を＋1する**記録**画面もありますが、**現在のナビでは運営ロール向けに「記録」タブが表示**され、生徒ナビからは直接開けません。デモを確認する場合は運営アカウントで「記録」を開くと説明文が表示されます。

---

## 4. 生徒向け：各画面の操作と挙動

### 4.1 ホーム

| できること | タイミング・挙動 |
|------------|------------------|
| 早期投稿スプリントなどのバナー | 条件を満たすと表示。タップで案内どおりの画面へ誘導されることがあります。 |
| 学習進捗・投稿サマリー | データがあるとカード表示。更新は主に**投稿タブ**や**ロードマップ**での操作後に反映されます。 |
| 今日の予定 | **スケジュール**の予定・セッション由来の予定などが並びます。項目の✓で完了にできます。 |
| 予定をもっと見る系 | **スケジュール**タブへ切り替わります。 |
| 目標（ビジョン）カード | マイページで目標を入れていると表示。編集はマイページからです。 |

### 4.2 投稿

| できること | タイミング・挙動 |
|------------|------------------|
| 月間ヒートマップ | 月の投稿状況を俯瞰。日付をタップすると詳細シートが開きます。 |
| **投稿予定日** ＋ **新しい投稿** | 選んだ日がその投稿の**予定日**になります。作業開始日は「予定日から○日さかのぼり」（**マイページの投稿ペース**で変わる）で自動設定され、今日より前にはなりません。 |
| 投稿カードの切替（#1 #2 …） | 進行中の投稿が複数あるとき、ピルで選択。工程ガントが切り替わります。 |
| 工程のチェック | 各ステップの完了が進捗％に反映。**100%**になると「投稿完了（記録前）」としてスケジュール等に関係します。 |
| **まとめて記録** | 工程100%だがまだカレンダーに記録していない投稿があるとき表示。まとめてカレンダー記録用の予定が作られます。 |
| 新規作成・工程更新後 | **スケジュール・ホーム・ロードマップ**も再計算され、セッションの「投稿」カウントやチェック連動が更新されることがあります。 |

**よくある流れ**: 予定日を決める → 投稿タブで工程を進める → 100%になったら必要に応じてまとめて記録。

### 4.3 ロードマップ

上部の **マイルストーン / スターター / セッションタスク** で表示が切り替わります。

#### マイルストーン

- コースの大きな節目。達成記録・期限のイメージを確認できます。

#### スターター

- 初期プログラムのステップをチェック形式で進めます。

#### セッションタスク（セッションカード）

1枚のカードが **1回分のコーチングセッション** を表します。

| 要素 | 説明 |
|------|------|
| **実施日** | ✎ から変更可能。カレンダー上のセッション予定とも連動します。 |
| **目標 / 投稿 / チェック** | **実施日〜次回セッションまで**の区間で集計。**目標**は「次回まで」タブの本数。**投稿**はその区間で**投稿タブの工程が100%完了した本数**。**チェック**はタスクの「n投稿」スロットの完了数／目標本数。 |
| **タスク** タブ | 上段に **プルダウン・日付・追加**。その下にタスク一覧。**投稿**を選ぶと `1投稿` `2投稿`…のように連番で追加されます（目標スロット用の行とは別扱い）。 |
| **次回まで** タブ | 次回セッションの日時、目標本数（0〜52）、**自分の目標**（テキスト）、**MGから**（閲覧のみ：次回までの共有／メッセージ）。➤で保存。 |
| **セッションと投稿カードを同時に作成**（次の回が未登録のとき） | 次の回の**実施日**と**その日までに増やす投稿数**を入れて実行 → **次のセッションカード**ができる／**投稿タブに工程カード**が並ぶ／**直前のセッションの目標本数**が入力値に合わせて **1投稿…** タスクが並ぶ。**画面はロードマップのまま**（投稿タブに自動で飛びません）。 |
| 投稿だけ増やすカード | 次のセッションがすでにある場合のみ表示されるブロックから、日付と本数で投稿カードだけ追加できます（こちらも**ロードマップに留まる**設定です）。 |

**タイミングの例**

- **投稿タブで工程を100%**にすると → 対応する区間のセッションで、**チェック**が自動で付くことがあります（区間内の並びとスロット番号で対応）。
- **目標本数**を変える → **n投稿**の行数が合わせて増減します（手動で付けた「n投稿」は目標の並び替えに巻き込まれないよう分離されています）。

### 4.4 スケジュール

| サブタブ | 内容 |
|----------|------|
| **スケジュール** | 月カレンダーと、日付別の予定一覧。日付をタップで選択。予定の完了チェックができます。投稿予定・投稿完了（記録前/済）も表示されます。 |
| **メモ** | 学び・気づきのメモ（カテゴリ別／日付別）。 |

**挙動の例**

- 予定行の **投稿へ** 系ボタン → **投稿**タブへ移動し、その投稿が選ばれた状態で開きます。
- 投稿の予定日変更の本体は**投稿タブ**側の操作が中心です（スケジュール上は確認・完了が中心、という位置づけ）。

### 4.5 マイページ

| できること | 挙動 |
|------------|------|
| プロフィール編集・写真 | 名前、SNS URL、クラス、開始日／卒業日など。 |
| 目標（ビジョン） | 編集画面へ。ホームのカード表示に反映されます。 |
| 実績（ヶ月目・累計投稿など） | 記録されたデータから集計表示。 |
| **投稿ペース** | 「予定日から何日前から作業開始とみなすか」に関わります（投稿新規作成時）。 |
| 定期タスク | 繰り返しの予定に近い設定。 |
| メモへ | **スケジュール**のメモサブタブへ誘導します。 |

## 5. 運営向け（マネージャー・リーダー・管理者など）

下部ナビは **ホーム・タスク・記録・（チーム）・マイページ** 構成になります（生徒専用タブは非表示）。

### 5.1 ホーム

- 期限が近いタスクを最大5件ほど表示。スワイプ等の操作は実装に従います。

### 5.2 タスク

| 内容 | 挙動 |
|------|------|
| 生徒にタスク追加 | 生徒・種類・目標回数・期限などを入力して追加。 |
| 一覧 | 権限の範囲のタスクが表示されます。 |
| **管理者** | スターター工程、カレンダー予定プリセット、セッションタスクのプルダウン文言などの編集ブロックが追加で出ます。 |

### 5.3 記録

- **生徒でない**場合: 「生徒アカウントで…」の説明カードのみ。
- **生徒の場合**（コード上）: 今日のタスクを＋1する大きなボタンなど（※現行ナビでは生徒はこのタブを持たない構成です）。

### 5.4 チーム（運営のみ表示）

- 担当生徒の進捗サマリー・今日の予定チップなど。生徒を選ぶと詳細寄りの表示に切り替わります。

### 5.5 マイページ

- 自分のプロフィール中心（生徒向けのSNSブロック等はロールに応じて省略されます）。

## 6. 画面遷移が自動で起きる主なケース

| きっかけ | 遷移・更新 |
|----------|------------|
| ログイン直後 | **ホーム** |
| スケジュールから「投稿を開く」 | **投稿**タブへ |
| 一部のホーム・バナー操作 | スケジュールや投稿へ（実装箇所によります） |
| ロードマップの「セッション＋投稿カードを同時に作成」「投稿カードだけ作成」 | **投稿タブへは飛ばず**、その場で再描画（現在の仕様） |
| タスク追加フォームからの一括投稿作成（別経路） | **投稿**タブへ移動する処理が残っている箇所があります |

## 7. 用語の整理（セッションと投稿）

- **セッションカード**: 面談◯回目。実施日・次回までの入力・セッション単位のタスクがあります。
- **投稿カード（投稿タブ）**: 1本の投稿について、台本・撮影などの**工程**を進める単位です。
- **目標本数（次回まで）**: そのセッション期間で「何本仕上げるか」。タスクに **1投稿 2投稿…** が並びます。
- **「投稿」（ヘッダ数字）**: 期間内で**工程100%になった本数**のイメージで見るとわかりやすいです。

---

## 8. 困ったとき

- **表示がおかしい** → いったんログアウト／再ログイン、またはブラウザの更新。
- **データが消えた** → localStorage がクリアされていないか確認（デモは端末依存）。
- **ファイル構成** → アプリ本体は `coaching-mobile/index.html` または `jouzu-demo-share.html`（単一HTMLビルド）など。開発時は `app.js` / `styles.css` を編集し、共有用HTMLは `build-share.sh` で再生成する運用です。
